# Process — v4 (archived)

## Why superseded

v4's gross lead time analysis was wrong. It computed "agent work time only"
(~49–77 min across two slices) and identified the tester as the constraint.

The correct gross lead time is wall-clock time from idea → running in prod,
including ALL waits: human gate pauses, system idle time, overnight periods.

Correct figures for ox (2 slices):
- Total wall clock: ~24.5 hours
- Total agent work: ~2.1 hours
- Wait time: ~22.4 hours = **91% of gross lead time**

The real constraint is **human gate latency** — each blocking gate is a
potential multi-hour or overnight wait. Agent processing time (including the
tester) is a secondary concern under this correct definition.

## v4 changes and their scores

| Change | Anticipated | Score |
|--------|-------------|-------|
| Tester scope (no re-implementing exhaustive checks) | Tester median < 1200s | Addresses agent work time but wrong layer — gate latency dominates. Still worth keeping. |
| Parallel next-slice planning | Remove architect from critical path | Correct direction — but the real win is reducing blocking gate count, not agent parallelism. |
| Auto-approve go/no-go for local-only green builds | Remove one human wait | Correct and significant — one fewer potential overnight wait. |

All three changes carry forward to v5. The frame shifts from "agent speed" to
"gate count and non-blocking progress."
