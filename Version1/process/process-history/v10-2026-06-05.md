See `process-current.md` for v11.

## What changed in v10 (from v9)

Added security review auto-accept rule (§8a): when solution-architect's delta
explicitly concludes "no new attack surface, no new data flow, no new trust
boundary", orchestrator confirms and auto-accepts — no human gate.

Anticipated DORA effect: gate 3 human wait drops to zero on slices with no
infra change.

   condition must include smoke test correctness for "surface-change" phases.
   New rule added to §22 (engineer done condition — surface changes).

   readiness legend, a lighter architect review pattern is formalised to reduce
   the planning phase from ~15 min toward ~5 min.
