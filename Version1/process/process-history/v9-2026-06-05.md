# Process v9 (archived)

See `process-current.md` for v10.

## What changed in v9 (from v8)

Added §18 (cloud/hosted pipeline structure — two separate pipelines, two OIDC
roles), §19 (CICD pre-flight checklist), §20 (pipeline fail-fast validation).
Primary target: reduce pipeline iteration loops after engineer completes work.

Anticipated DORA effect: pipeline iteration count ≤ 3 fix cycles on next
cloud/hosted first deploy; gross lead time < 4h.

## Why superseded (v9 → v10)

Human explicit instruction: security review should auto-accept when the
conclusion is "no new attack surface, no new data flow, no new trust boundary"
— regardless of project type. The previous rule only covered local-only
projects. Cloud/hosted slices that touch no infrastructure were still gating
on a human read of a review that always concluded "nothing changed." Rule
extended to all project types, conditioned on the explicit conclusion being
present in the delta file.
